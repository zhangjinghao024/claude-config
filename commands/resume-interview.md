# 简历项目面试题库生成技能

你的任务是基于候选人的简历和项目经历，生成一套完整的、有深度的面试题库，每题附带参考答案要点、追问方向和难度标记。

## 使用流程

### 第一步：获取简历信息

直接从 /Users/zhangjinghao/Desktop/简历/ 目录读取简历文件，提炼以下关键信息：

- 项目名称、技术栈、项目规模
- 用户在项目中的具体职责和贡献
- 量化成果（性能数据、业务指标等）
- 技术亮点和难点

### 第二步：生成题库

按项目和维度组织题目。每个项目覆盖以下四个维度，每个维度按照前端、移动端情况酌情出题。

#### 输出格式

每道题使用以下结构：

```
**Q[编号]**: [问题]
- 考察点: [这道题在考什么]
- 参考答案: [答案要点，2-5条]
- 追问方向: [面试官可能的追问]
```

### 题目设计四维度

#### 维度一：项目细节追问

围绕简历中每个项目，生成架构设计、技术选型、难点攻克、成果量化等方面的问题。

重点关注：
- 项目的整体架构和用户负责的模块
- 为什么做出某个技术决策（选型 trade-off）
- 遇到的最大技术挑战及解决方案
- 量化成果背后的具体实现细节（如"秒开率提升39.2pp"到底是怎么做到的）
- 如果重做，有什么会改进的地方

#### 维度二：前端基础

结合项目中用到的技术出题，不要出纯八股文，尽量与项目场景结合。

示例方向：
- 闭包在项目中的实际应用场景
- 事件循环与异步编程在性能优化中的角色
- 虚拟 DOM diff 算法与渲染优化
- 浏览器/RN 渲染流程与秒开率优化的关系
- TypeScript 类型系统在工程中的价值

#### 维度三：移动端/跨平台专项

React Native 架构原理、跨平台适配方案、原生桥接、性能调优等。

示例方向：
- RN 的 JS Bridge / JSI 通信机制及性能瓶颈
- 跨平台适配策略（iOS/Android/鸿蒙差异处理）
- 原生模块开发与桥接层设计
- 热更新方案与包体积优化
- 折叠屏适配的技术方案
- RN 新架构（Fabric、TurboModules）

#### 维度四：系统设计 & 性能优化

结合项目规模和业务场景出综合设计题。

示例方向：
- 设计一个高并发场景下的前端方案（如抢票）
- 页面秒开率优化的完整方案设计
- 大型 RN 项目的状态管理架构
- BFF 层的设计原则与实践
- 前端监控与错误上报体系设计
- MCP 协议在企业工具链中的应用设计

## 题目设计原则

- 题目必须紧密围绕用户简历中的真实项目，避免泛泛的通用题
- 参考答案要结合用户的项目背景，不要给出与用户经历无关的通用答案
- 追问方向要有逻辑链条，体现从浅到深的递进关系
- 对于量化成果（如秒开率提升39.2pp），一定要设计追问具体实现路径和度量方式的题目
- 每个项目至少有 2 道"如果重做/扩展"类的开放题，考察反思和成长能力
